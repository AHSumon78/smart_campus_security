Smart Campus Security AI System

Supervisor-friendly project overview

4. AI Face Recognition Workflow

  1. User opens Face Check or Live Detection.
  2. System captures image from camera/upload.
  3. Image is sent to the AI recognition service.
  4. AI extracts face embedding and compares with stored student embeddings.
  5. If match is found, system returns student details, confidence, department, and access status.
  6. Every attempt is saved in Access Log.
  7. If the person is banned or suspicious activity is detected, a security alert is created.

5. Live AI Camera Detection

The live camera system can detect multiple faces in one frame, identify known students, mark unknown faces, and highlight banned persons.

6. Dashboard and Analytics

Management can see total students, generated embeddings, today's access, pending photo requests, department-wise chart, and access source distribution.

7. Access History and Audit Log

Every success/denied attempt is recorded with student name, ID, source/location, date, time, search/filter, print PDF, and cleanup option.

8. Student Management

The system supports student add/edit/delete/search, photo storage, department data, active/banned status, and embedding generation.

9. Photo Update Review

Staff can submit a new photo request, and admin can compare old vs new photo before approving or rejecting. This prevents unauthorized photo replacement.

10. Attendance Management

The attendance module supports course/session based live attendance, upload photos/video, manual mark present, percentage calculation, and Excel/PDF export.

11. User Management

Admin can create/edit/delete system users and assign roles like Staff, Manager, Security Guard, Registration Desk, and Especial.
